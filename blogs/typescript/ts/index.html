# 🍂 Quick Start

TIP

这里假定你已经安装了node环境

# 初始化 package.json

  1. 创建工程目录，并进入此目录

  2. 执行初始化命令（最终会在工程根目录生成 package.json）

    npm init
    1

  3. 根据提示修改配置，也可以全部使用默认配置（后期可在package.json中手动修改）

# 安装typescript本地开发依赖

npm install typescript --save-dev
1

安装typescript依赖后，可以使用tsc命令执行初始化、编译等行为

TIP

在终端输入tsc --all可以查看所有tsc命令

# 初始化tsconfig.json

tsc --init
1

执行此命令后将会在项目根目录生成tsconfig.json

目录结构：

.
├── package.json
└── tsconfig.json
1
2
3

# 编译

安装typescript依赖后，我们就可以使用tsc命令编译.ts文件了。

  • 编译整个工程
    • 单纯调用tsc命令，编译器会从当前目录逐级向上搜索tsconfig.json文件，并根据其配置进行编译。
    • 调用tsc --project projectName（或tsc -p projectName）可以指定一个包含tsconfig.json的目录，并根据其配置进行编译
  • 编译指定.ts文件
    • 编译单个文件：tsc file.ts
    • 编译多个文件：tsc file.ts file1.ts file2.ts
  • 自动编译
    • 监听文件并自动编译：tsc --watch（或tsc -w）
  • 你可以通过编译参数改变默认编译配置
    • 比如：tsc file.ts file2.ts --outDir dist（将dist作为编译生成的.js的输出目录）

TIP

在终端输入tsc --all可以查看所有tsc命令。

tsc --all
1

# 使用node测试编译后生成的.js文件

node xx/xx/x.js
1

# 使用ts-node直接调用.ts文件

node只能调用.js文件，我们可以安装ts-node本地开发依赖，从而跳过编译直接调用.ts文件

  1. 首先安装ts-node本地开发依赖

    npm install ts-node --save-dev
    1

  2. 直接调用.ts文件

    ts-node xxx/xxx/x.ts
    1

此时一个最最纯净的ts项目就创建完成了。
